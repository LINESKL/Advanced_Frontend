**Что получил:** Пример функции `loader`, которая извлекает `params.id`, и способ получения этих данных в компоненте через `useLoaderData()`.

### Промпт 2:
"Как реализовать сортировку списка с помощью useSearchParams, чтобы состояние сортировки сохранялось в URL (например, ?sort=asc)?"

**Что получил:** Логику работы с методом `searchParams.get('sort')` и функцию `setSearchParams` для обновления строки запроса при клике на кнопку.

### Промпт 3:
"Как создать динамическую ссылку <Link> на основе ID объекта, используя шаблонные строки?"

**Что получил:** Синтаксис `` <Link to={`/courses/${course.id}`}> ``, который позволяет генерировать пути динамически для каждого элемента списка.

## 3. Модификация и проверка
- **Типизация:** Описал интерфейс `Course` и строго типизировал возвращаемые данные из лоадера, избегая использования `any`.
- **Валидация:** Добавил проверку в `loader`: если курс с заданным ID не найден, выбрасывается ошибка, которую перехватывает `errorElement`.
- **Логика:** Самостоятельно реализовал функцию сортировки массива объектов по алфавиту в зависимости от текущего значения в URL.
